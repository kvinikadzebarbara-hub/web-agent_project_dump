# Web-Agent

## Описание: 
Web-Agent — это кроссплатформенный C++ агент, который связывает веб-интерфейс с локальными программами на клиентской машине. При нажатии кнопки в веб-приложении агент запускает нужную программу, обрабатывает данные и отправляет результат на сервер.
Проект демонстрирует применение современных возможностей C++: шаблоны, многопоточность, RAII, а также интеграцию популярных библиотек.

## Функционал:
Веб-агент регистрируется на сервере, периодически опрашивает его на наличие задач, получает инструкции, запускает указанные программы на локальном компьютере, собирает результаты выполнения (код возврата, вывод, файлы) и отправляет их обратно на сервер. Все действия записываются в лог. Поддерживается многопоточное выполнение задач и работа на Windows, Linux и macOS.

## Поддерживаемые платформы
Windows (Windows 10 / Windows 11),
Linux (Ubuntu 20.04+, Debian 11+, Fedora 38+, CentOS 8+, RHEL 8+, Arch Linux),
macOS (macOS 12 (Monterey) и новее)

#Руководство пользователя:
## 📖 Руководство пользователя: Работа с задачами

Агент получает задания через веб-интерфейс. В поле **«Опции»** указываются параметры в зависимости от выбранного типа задачи:

| Тип задачи | Назначение | Пример ввода в «Опции» | Что делает агент |
|------------|------------|------------------------|------------------|
| `TASK` | Выполнение команды | `echo "Hello"`, `ls -la`,  | Запускает в системной оболочке, возвращает stdout/stderr и код выхода |
| `FILE` | Загрузка файла на сервер | `config.json`,  | Ищет файл в `task_directory` и `result_directory`, отправляет содержимое |
| `CONF` | Изменение настроек | `poll_interval_sec=30`,  | Обновляет конфигурацию в памяти. Некоторые параметры требуют перезапуска |
| `TIMEOUT` | Лимит времени выполнения | `60`, `300` | Устанавливает таймаут (сек) для следующей задачи `TASK` |

### 📌 Правила ввода
- Для `CONF` используйте строго формат `ключ=значение` **без пробелов** вокруг `=`.
- Для `FILE` указывайте относительные пути от рабочих директорий. Абсолютные пути и `..` блокируются в целях безопасности.
- Команды `TASK` выполняются с правами пользователя, запустившего агента.
- Пустое поле «Опции» для `TASK` выполнит пустую команду (успешный статус, но без вывода).

## 📊 Логирование и мониторинг

Все события записываются в файл, указанный в `config.json`.

###  Просмотр в реальном времени
```bash
# Linux/macOS
tail -f ./agent.log 

# Windows PowerShell
Get-Content .\agent.log 
```

### 🔄 Уровни логирования
Доступные уровни: `trace`, `debug`, `info`, `warn`, `error`, `critical`.  
Для отладки установите `"log_level": "debug"` в `config.json` и перезапустите агент. Не используйте `debug` в продакшене: логи могут содержать чувствительные данные.

## 🧪 Тестирование и разработка

Проект использует **Google Test** для модульного тестирования.

```bash
# Сборка тестов
cmake .. -DCMAKE_BUILD_TYPE=Debug
cmake --build . --target web_agent_tests

# Запуск
ctest --output-on-failure
# или ./web_agent_tests
```

📁 Тесты находятся в директории `tests/`. При добавлении новой логики покрывайте её юнит-тестами. Сетевые вызовы тестируются через моки, многопоточные модули — изолированно.

## 🛠️ Частые проблемы

| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| Статус задачи долго `WAIT` | Агент не запущен или неверный `uid`/`access_code` | Проверьте `agent.log`, сверьте `config.json` с настройками сервера |
| `Unknown config parameter` | Опечатка или пробелы в задаче `CONF` | Используйте точные имена из конфига, формат `key=value` |
| Файл не отправляется (`FILE`) | Файл не найден в рабочих папках | Проверьте путь, права доступа (`chmod 644`), убедитесь в существовании файла |
| Таймаут задачи `TASK` | Команда выполняется дольше установленного лимита | Увеличьте таймаут через задачу `TIMEOUT` |
| Агент падает при старте | Отсутствуют зависимости или нет прав на запись логов | Установите пакеты из раздела «Быстрый старт», проверьте права на директорию |

## 🛡️ Безопасность

⚠️ **Важно:** Агент использует `std::system()` для выполнения команд типа `TASK`.
- Запускайте агент под пользователем с минимальными необходимыми правами.
- Ограничьте доступ к веб-интерфейсу сервера (аутентификация, IP-фильтры, HTTPS).
- Не передавайте команды с повышенными привилегиями или конфиденциальные данные через задачи.
- Для продакшена рекомендуется добавить валидацию команд и белый список разрешённых утилит в `TaskExecutor.cpp`.
- Регулярно ротируйте `access_code` в конфигурации.
